# DAX — Frontend Engineering Direction (Light Mode)

# Purpose

This document defines the frontend engineering philosophy and implementation direction for the DAX light mode experience.

The frontend must not feel:
- template-based
- component-generated
- startup-oriented
- utility-framework driven
- visually generic

The engineering must support:
- cinematic presentation
- immersive interaction
- artistic composition
- atmospheric rendering
- luxury motion systems
- premium visual execution

Frontend engineering is part of the art direction itself.


---

# Core Engineering Philosophy

The website should be engineered like:
- a digital experience
- an immersive installation
- a cinematic interface
- a premium visual product

Not like a standard marketing website.

Every technical decision should reinforce:
- atmosphere
- smoothness
- visual sophistication
- emotional immersion
- spatial depth


---

# Technology Stack Direction

Preferred stack:

- Next.js
- React
- Tailwind CSS (heavily customized)
- Framer Motion
- GSAP
- Lenis Smooth Scroll
- Three.js (when appropriate)
- WebGL enhancements (subtle and optimized)

The stack must support:
- cinematic rendering
- fluid animation
- GPU acceleration
- immersive responsiveness


---

# Tailwind Philosophy

Tailwind should not feel visible.

Avoid:
- default Tailwind appearance
- repetitive utility styling
- obvious framework aesthetics
- startup component patterns

Use Tailwind only as:
- an implementation utility
- a low-level styling system

The final result must feel completely custom crafted.


---

# Component Philosophy

Components should feel:
- art directed
- spatially aware
- visually integrated
- cinematic
- atmospheric

Avoid:
- generic reusable startup blocks
- dashboard component mentality
- repetitive card systems

Components should feel handcrafted.


---

# Rendering Philosophy

Rendering should prioritize:
- smoothness
- visual softness
- atmospheric layering
- cinematic depth
- responsive performance

The experience should feel physically smooth and visually immersive.


---

# GPU Acceleration Philosophy

Use GPU acceleration whenever appropriate.

Prefer:
- transform-based animation
- opacity transitions
- GPU-friendly blur systems
- optimized motion rendering

Avoid:
- layout thrashing
- repaint-heavy effects
- unoptimized filter abuse

Motion should remain luxurious and stable.


---

# Animation Engineering

Animations should feel:
- cinematic
- fluid
- premium
- physically believable

Use:
- Framer Motion for UI choreography
- GSAP for cinematic sequencing
- Lenis for smooth immersive scrolling

Animation systems should remain coordinated and emotionally paced.


---

# Scroll Engine Philosophy

Scrolling is part of the cinematic experience.

The website should use:
- smooth scrolling
- inertia-based movement
- atmospheric pacing
- layered parallax
- cinematic section progression

Scrolling should feel immersive and luxurious.


---

# Blur Engineering

Blur is critical to the atmosphere.

Use:
- backdrop-filter carefully
- layered blur systems
- atmospheric diffusion
- optimized blur rendering

Avoid:
- excessive heavy blur
- performance-breaking filters
- uncontrolled blur stacking

Blur should feel soft and effortless.


---

# Gradient Engineering

Gradients should behave like environmental lighting.

Use:
- radial gradients
- layered opacity gradients
- atmospheric blending
- soft luminosity transitions

Avoid:
- obvious CSS gradient aesthetics
- startup gradient clichés
- oversaturated color transitions

Gradients should feel cinematic.


---

# Lighting Implementation

Lighting should be built through:
- layered gradients
- atmospheric glow systems
- radial diffusion
- environmental overlays
- cinematic blur interaction

Lighting should feel embedded into the environment.


---

# Image Rendering Philosophy

Images should integrate deeply into the interface.

Use:
- gradient masking
- edge blending
- atmospheric overlays
- cinematic color grading
- layered opacity systems

Images should never feel isolated.


---

# Typography Engineering

Typography rendering should prioritize:
- cinematic scale fluidity
- elegant responsiveness
- optical spacing
- premium readability
- compositional integration

Typography must feel spatial and refined across all screen sizes.


---

# Responsive Philosophy

Responsiveness should preserve:
- emotional scale
- cinematic spacing
- atmospheric immersion
- compositional hierarchy

Avoid:
- collapsing layouts into generic mobile stacks
- destroying visual rhythm
- losing cinematic feeling

The mobile experience must remain premium.


---

# Motion Performance Philosophy

Motion must remain:
- smooth
- performant
- stable
- immersive

Avoid:
- dropped frames
- stuttering
- excessive simultaneous animation
- animation overload

Luxury motion feels effortless.


---

# Section Engineering Philosophy

Sections should not feel isolated.

Use:
- shared environmental systems
- continuous atmosphere
- lighting continuity
- compositional blending

The page should behave as one immersive environment.


---

- improves immersion

- strengthens visual storytelling

- unnecessary complexity

Technology should support emotion.

Not overshadow it.
